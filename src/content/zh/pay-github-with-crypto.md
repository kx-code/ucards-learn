## 为什么要用加密货币支付 GitHub？

但支付这些订阅并不总是那么容易。许多开发者是自由职业者或外包人员，没有公司卡。有些人所在地区限制了国际卡片支付。还有些人更愿意直接用已有的加密货币支付，而不是通过银行转换。

加密充值的虚拟卡为你提供了一张 GitHub 接受的 Visa 卡，无需 KYC、无需银行账户。

## 如何用加密货币支付 GitHub：详细步骤

### 第一步：创建 uCards 虚拟卡

访问 [ucards.uk](https://ucards.uk) 注册。无需身份验证或上传文件。虚拟 Visa 卡会立即生成，包含卡号、有效期和 CVV。

### 第二步：用加密货币充值

选择你偏好的加密货币充值：

- **USDT（TRC-20）** — 推荐大多数用户使用。TRON 网络转账费仅 $1，3 分钟确认
- **ETH（ERC-20）** — 适合大额充值。网络费用随拥堵程度变化
- **BNB（BEP-20）** — BSC 网络费用最低，通常不到 $0.50

GitHub Pro 月费 $4，充值 $10-15 可覆盖数月。Copilot 月费 $10，充值 $15-20。Copilot 年费 $100，充值 $110-115。

### 第三步：将卡片绑定到 GitHub

1. 登录 [github.com](https://github.com)，点击头像
2. 进入 **Settings** > **Billing and plans**
3. 点击 **"Add a payment method"** 或升级方案
4. 选择 **"Credit or debit card"**
5. 输入 uCards 卡号、有效期和 CVV
6. 填写账单地址 — 任意有效地址即可
7. 点击 **"Save"** 确认

GitHub 会发起一笔 $1 的临时授权来验证卡片。这是标准操作，几天内会释放。

### 第四步：升级方案

卡片保存后，选择你想要的方案：

- **GitHub Pro**：点击 "Upgrade to Pro"
- **Copilot**：前往 copilot.github.com 订阅
- **GitHub Team**：在组织设置中升级

扣款即时处理，升级功能在几秒内激活。

GitHub 全球可用，但各国的支付处理方式不同。Visa 卡可以绕过地区支付限制。在银行受限国家的开发者也能使用高级工具。

GitHub 会发送邮件通知并重试扣款。多次失败后，账户会降级为免费版。你的代码和仓库完好无损，但高级功能会停用，直到恢复付款。

## GitHub + 加密货币：常见问题

**可以用加密货币支付 GitHub Sponsors 吗？**
GitHub Sponsors 使用你保存的付款方式。如果虚拟卡设为默认，赞助付款会从虚拟卡扣款。

**GitHub 会知道我在用加密货币吗？**
不会。GitHub 处理的是标准的 Visa 卡交易。加密货币充值层对 GitHub 完全不可见。

**一张卡可以支付多个 GitHub 组织吗？**
可以。你可以将同一张 uCards 卡绑定到多个 GitHub 账户或组织。只需确保余额能覆盖所有订阅。

**如何从月付切换到年付？**
在 GitHub 计费设置中切换计费周期。GitHub 会从你的卡片扣除年费金额，并调整下次续期日期。
